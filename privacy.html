Privacy Policy

Personal MCP Tools · Last updated 11 July 2026

Personal MCP Tools (“the tools”) is a collection of personal, open-source Model Context Protocol servers run by their author on their own computer to work with the author's own Google account. This policy explains how they handle Google user data.

Who this covers

The tools are single-user and self-hosted. They have no other users, no backend service, and no shared infrastructure. The only Google account they access is the author's own.

Google data the tools access

Each tool requests only the minimum Google OAuth scopes needed for its function. The scopes currently in use are the Gmail scopes below, used to manage labels, filters, and inbox rules:

  • gmail.labels — Gmail labels
  • gmail.modify — applying and removing labels on messages and threads
  • gmail.settings.basic — Gmail filters and basic settings

The tools do not request full-mailbox access and do not read the content of email messages — only message metadata and short snippets needed to preview an action. If tools for other Google services are added over time, each will likewise request only the minimum scopes it needs, and this policy will be updated to describe them.

How data is used

Data is used solely to perform the operations the author explicitly requests. It is not used for advertising, analytics, profiling, or any secondary purpose.

Storage and sharing

  • OAuth tokens are stored encrypted on the author's local machine and never transmitted anywhere except directly to Google's official APIs.
  • No Google user data is stored on any server or shared with any third party.
  • The tools do not sell, rent, or transfer any Google user data. Their use of Google data complies with the Google API Services User Data Policy, including the Limited Use requirements.

Data retention and deletion

The tools keep no copy of Google user content. The only persisted item is the encrypted OAuth token on the author's computer, which can be deleted at any time by removing the local token file. Access can be revoked at myaccount.google.com/permissions.

Contact

These are personal, single-user applications. Questions about this policy can be raised through the project's GitHub issues page.
